# 万象绘卷

> 全能故事创作与交互体验工具箱

## 功能概览

- **56种文风切换**：从殿堂文学到网文爽文，一键切换
- **网文创作全流程**：黄金三章、细纲生成、平台适配
- **交互式故事模拟**：实时互动、角色扮演
- **异世界人生模拟**：回合制人生、代代相传、后台成书
- **剧情管理系统**：长篇故事管理、大总结系统
- **异世界人生模拟**：回合制人生体验、世界观随机生成、代代相传、自由行动AP系统、后台成书
- **拆书融合创作**：多源小说融合创作新作品
- **小说档案馆**：爬取、分析、存储小说素材

## ⚠️ 透明声明

### 爬虫脚本说明

本技能包含以下爬虫脚本（位于 `scripts/` 目录）：

| 脚本 | 目标站点 | 用途 |
|------|----------|------|
| `crawl_novel_index.py` | novel.transchinese.org | 爬取分类索引与下载txt |
| `crawl_jh_novel.py` | 刺猬猫、起点等多平台 | 聚合小说搜索下载 |
| `crawl_fq_novel.py` | fanqienovel.com | 番茄小说搜索下载 |

**重要声明**：
- 所有爬虫脚本**仅供个人学习参考使用**
- 用户需自行确认目标站点的许可与版权合规，**使用者自行承担版权责任**
- 爬取内容不得用于商业用途
- 建议限制爬虫范围与频率，遵守 robots.txt

**数据流向说明**：
- ✅ **仅下载**：脚本只从目标站点下载内容到本地
- ❌ **不上传**：脚本不会向任何外部端点发送数据
- 📁 **本地存储**：所有下载内容仅存储在用户本地设备
- 🔧 **手动运行**：脚本需要用户手动执行，不会自动运行
- ⚠️ **无自动管道**：爬虫脚本与创作流程之间没有自动连接，不会程序化触发

### 后台成书说明

"后台成书"是指：在交互式故事/人生模拟过程中，AI将已发生的剧情以小说文体重新组织，**作为回复内容展示给用户**。
- 这不是自动磁盘写入或后台文件操作
- 用户需通过 `#小说` 命令主动查看
- 存储由用户自行决定（复制/保存/导出）
- **不会在用户未操作的情况下自动写入文件**

### 数据存储说明

本技能提供**本地数据存储系统**，用于持久化故事、角色、世界状态等数据：

#### 存储架构
- **SQLite 数据库**：`data/wanxiang.db` - 存储结构化数据（故事元信息、角色、事件、世界状态等）
- **JSON 文件**：`stories/[剧本名]/` - 存储小说文本、配置文件等
- **Markdown 文件**：`stories/[剧本名]/novel.md` - 存储生成的小说内容

#### 目录结构
```
data/
├── wanxiang.db          # SQLite 数据库

stories/
├── interactive/         # 交互式故事存储
├── templates/           # 模板存储
└── [剧本名]/            # 人生模拟剧本存储
    ├── novel.md         # 小说文本
    ├── data.json        # 游戏状态
    ├── legacy.json      # 前代角色数据
    └── world_state.json # 世界状态
```

#### 初始化
运行以下命令初始化数据库和目录结构：
```bash
python scripts/init_database.py
```

#### 存档导出
运行以下命令导出存档为 Markdown 格式：
```bash
python scripts/save_exporter.py
```

**存档输出格式**：
- **JSON 文件**：`stories/[剧本名]/data.json` - 中间数据格式
- **Markdown 文件**：`stories/[剧本名]/save.md` - 最终存档格式（用户可读）

**存档模板**：`saves/templates/`
- `life_simulation.md/json` - 人生模拟存档模板
- `interactive_story.md` - 交互式故事存档模板
- `minecraft_survival.md/json` - Minecraft生存存档模板
- `fusion_project.md` - 拆书融合项目模板

### 隐藏楼层说明

"隐藏楼层"（`#隐藏楼层`）是剧情管理机制，用于：
- 记录剧情伏笔和创作者备注
- 数据存储在 `hidden_floors` 表中
- 支持标记为"已解决"状态
- 可跨会话持久化

### 去AI化说明

"去AI化"（去AI痕迹）是写作质量优化技法，目的是让文本读起来更自然、更像人类手写风格：
- 这是文学创作领域的「润色」技法，与学术论文写作规范类似
- 目标是提升文本可读性和文学性，**不涉及AI检测技术**
- 本技能不包含任何AI检测规避工具
- 不建议也不支持使用去AI化功能来违反平台审核或内容政策

## 模块联动

所有模块不再独立，形成完整创作生态：

- **文风系统**（第二章）→ 交互故事/人生模拟/网文创作 的叙事风格引擎
- **创作引擎**（第五章）→ 人生模拟后台成书的去AI化处理 / 网文创作的质量保障
- **剧情管理**（第四章）→ 人生模拟的阶段大弧+连锁支线 / 交互故事的多线管理
- **交互式故事**（第三章）↔ 人生模拟（第十四章）两模式共享世界观+角色数据，可互相切换
- **拆书融合**（第十一章）→ 人生模拟的世界观素材来源 / 文风系统的参考库
- **小说档案馆**（第十三章）→ 人生模拟的成书存档 / 拆书分析的素材来源
- **质量控制**（第七章）→ 所有创作输出（网文/小说/后台成书）的审核把关
- **李诞七步**（第十五章）→ 人生模拟结局写作 / 网文开篇写作 的结构框架
