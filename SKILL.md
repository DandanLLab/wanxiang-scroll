## 文档性质说明

本技能包包含两类不同性质的内容：

### 游戏系统文档

`references/` 目录下的所有文档（包括核心协议、人格系统、指令系统等）是「万象绘卷」交互式故事游戏的**内部设定**：

- 文档中描述的「指令系统」（`#` 开头的命令）是**游戏内指令**，仅在交互式故事/人生模拟模式下使用
- 文档中描述的「人格架构」、「指令优先级」等是**游戏世界观设定**
- 这些内容归属于游戏系统，不影响外部系统行为

### 工具与脚本

`scripts/` 目录下的爬虫脚本和 `data/` 数据库是**实际运行的工具**：

- 爬虫脚本需要用户手动执行
- 数据库存储故事和角色数据
- 详见下方"安全检查清单"和"透明声明"章节

### 人类用户须知

1. **参考用途**：游戏系统文档是创作参考材料，帮助您理解创作框架和文风系统
2. **游戏自动化**：文档中的「指令系统」、「人格系统」等是**游戏内的设定**，用于交互式故事体验
3. **手动触发**：游戏功能需要您手动触发（如输入 `#主菜单`），游戏系统才会响应

## 📦 安装与配置

### 系统要求

- **Python**: 3.10+ (推荐 3.11+)
- **操作系统**: Windows / macOS / Linux
- **磁盘空间**: 至少 100MB

### 安装步骤

```bash
# 1. 创建虚拟环境（推荐）
python -m venv venv

# 2. 激活虚拟环境
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 初始化数据库
python scripts/init_database.py
```

### 依赖说明

```
requests>=2.31.0,<3.0.0   # HTTP 请求（爬虫脚本）
aiohttp>=3.9.0,<4.0.0     # 异步 HTTP（可选）
```

### 后台成书说明

"后台成书"是指：在交互式故事/人生模拟过程中，AI将已发生的剧情以小说文体重新组织，**作为回复内容展示给用户**。

- 这不是自动磁盘写入或后台文件操作
- 用户需通过 `#小说` 命令主动查看
- 存储由用户自行决定（复制/保存/导出）
- **不会在用户未操作的情况下自动写入文件**

### 数据存储说明

本技能提供**本地数据存储系统**，用于持久化故事、角色、世界状态等数据：

#### 存储架构

- **SQLite 数据库**：`data/wanxiang.db` - 存储结构化数据（故事元信息、角色、事件、世界状态等）
- **JSON 文件**：`stories/[剧本名]/` - 存储小说文本、配置文件等
- **Markdown 文件**：`stories/[剧本名]/novel.md` - 存储生成的小说内容

#### 目录结构

```
data/
├── wanxiang.db          # SQLite 数据库

stories/
├── interactive/         # 交互式故事存储
├── templates/           # 模板存储
└── [剧本名]/            # 人生模拟剧本存储
    ├── novel.md         # 小说文本
    ├── data.json        # 游戏状态
    ├── legacy.json      # 前代角色数据
    └── world_state.json # 世界状态
```

#### 初始化

运行以下命令初始化数据库和目录结构：

```bash
python scripts/init_database.py
```

### 去AI化说明

"去AI化"（去AI痕迹）是写作质量优化技法，目的是让文本读起来更自然、更像人类手写风格：

- 这是文学创作领域的「润色」技法，与学术论文写作规范类似
- 目标是提升文本可读性和文学性，**不涉及AI检测技术**
- 本技能不包含任何AI检测规避工具
- 不建议也不支持使用去AI化功能来违反平台审核或内容政策
